# E0103: Integer Literal Too Large

## Explanation

The lexer parsed an integer literal that does not fit in Nomo's current `i64`
literal range.

## Error Example

```nomo
package app.main

const TOO_BIG: i64 = 999999999999999999999999999999
```

## Correct Code

```nomo
package app.main

const VALUE: i64 = 9223372036854775807
```

## Fix

Use a smaller integer literal or represent the value with a domain-specific
type once that type is available.

## LSP Code Action

No automatic code action is currently provided.

# E0104: Invalid Character Literal or Reserved Word

## Explanation

The lexer reports this code for two tokenization errors that are currently part
of the stable v0.1 diagnostic surface:

- a character literal starts with `'` but does not close correctly;
- an identifier uses a word reserved for future Nomo versions.

## Error Example

```nomo
package app.main

fn main() -> void {
    let letter: char = 'a
}
```

Another example is using a future-reserved word as an identifier:

```nomo
package app.main

fn go() -> void {
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    let letter: char = 'a'
}
```

For reserved words, choose a different identifier:

```nomo
package app.main

fn run() -> void {
}
```

## Fix

Close the character literal with `'`, or rename identifiers that collide with
reserved future keywords.

## LSP Code Action

No automatic code action is currently provided.

# E0200: Missing Package Declaration

## Explanation

Every Nomo source file starts with a `package` declaration. The parser reports
this error when the first declaration is missing or malformed.

## Error Example

```nomo
fn main() -> void {
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
}
```

## Fix

Add `package <path>` as the first declaration in the file. In a project,
`src/main.nomo` normally uses the root package path from `nomo.toml`.

## LSP Code Action

No automatic code action is currently provided.

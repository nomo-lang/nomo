# E0201: Invalid Top-Level Declaration Order

## Explanation

The parser found a top-level token where only a declaration or the end of the
file is valid. This also happens when a declaration appears after top-level
script statements, because declarations must come first.

## Error Example

```nomo
package app.main

io.println("hello")

fn helper() -> void {
}
```

## Correct Code

```nomo
package app.main

fn helper() -> void {
}

io.println("hello")
```

## Fix

Move all declarations before top-level script statements, or remove a stray
`pub` / declaration marker that is not followed by a valid declaration.

No automatic code action is currently provided.

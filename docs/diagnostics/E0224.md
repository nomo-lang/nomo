# E0224: Invalid Struct Literal Field Separator

## Explanation

After each struct literal field, the parser expects either `,` for another
field or `}` to close the literal.

## Error Example

```nomo
package app.main

struct User {
    name: string
    email: string
}

fn make() -> User {
    return User { name: "Nomo" email: "hi@example.test" }
}
```

## Correct Code

```nomo
package app.main

struct User {
    name: string
    email: string
}

fn make() -> User {
    return User { name: "Nomo", email: "hi@example.test" }
}
```

## Fix

Add a comma between struct literal fields, or close the literal with `}` after
the final field.

No automatic code action is currently provided.

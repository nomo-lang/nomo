# E0236: Invalid Generic Type Argument Separator

## Explanation

After a generic type argument, the parser expects `,` for another type argument
or `>` to close the argument list.

## Error Example

```nomo
package app.main

struct Pair<T, U> {
    left: T
    right: U
}

fn make() -> Pair<i64 string> {
    return Pair { left: 1, right: "one" }
}
```

## Correct Code

```nomo
package app.main

struct Pair<T, U> {
    left: T
    right: U
}

fn make() -> Pair<i64, string> {
    return Pair { left: 1, right: "one" }
}
```

## Fix

Add a comma between generic type arguments, or close the type argument list with
`>`.

No automatic code action is currently provided.

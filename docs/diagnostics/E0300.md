# E0300: Expected Identifier

## Explanation

The parser expected an identifier or a dotted path segment but found another
token. This can happen in declarations, imports, type names, variable names, or
after `.` in a path.

## Error Example

```nomo
package app.main

import std.

fn main() -> void {
}
```

## Correct Code

```nomo
package app.main

import std.io

fn main() -> void {
}
```

## Fix

Add the missing identifier, remove the dangling `.`, or rewrite the declaration
so every name position contains a valid identifier.

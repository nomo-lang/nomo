# E0301: Unknown Name or Missing Import

## Explanation

The compiler could not resolve a referenced name in the current scope, imported
standard library modules, or project modules.

## Error Example

```nomo
package app.main

fn main() -> void {
    println("hi")
}
```

## Correct Code

```nomo
package app.main

import std.io.println

fn main() -> void {
    println("hi")
}
```

## Fix

Check spelling, define the missing item, or add the required import. For
standard library calls, prefer the concrete import suggested by the compiler.

## LSP Code Action

Yes. When the compiler can infer a concrete import, `nomo-lsp` exposes a quick
fix that inserts it.

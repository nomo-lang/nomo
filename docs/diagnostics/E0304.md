# E0304: Duplicate Callable or Constant Definition

## Explanation

A top-level callable or constant name is already defined. This code is used for
duplicate functions, methods after name lowering, interfaces, and constants.

## Error Example

```nomo
package app.main

fn helper() -> void {
}

fn helper() -> void {
}
```

## Correct Code

```nomo
package app.main

fn helper() -> void {
}

fn other_helper() -> void {
}
```

## Fix

Rename one declaration or remove the duplicate. For methods, ensure an impl
block does not define the same method twice for the same owner type.

## LSP Code Action

No automatic code action is currently provided.

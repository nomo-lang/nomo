# E0312: Type Namespace or Struct Field Conflict

## Explanation

This diagnostic covers two namespace conflicts:

- a user type conflicts with another type or a required standard-library type;
- a struct literal contains a field that does not exist on the target struct.

## Error Example

```nomo
package app.main

struct Point {
    x: i64
}

fn main() -> void {
    let point: Point = Point { x: 1, y: 2 }
}
```

## Correct Code

```nomo
package app.main

struct Point {
    x: i64
    y: i64
}

fn main() -> void {
    let point: Point = Point { x: 1, y: 2 }
}
```

## Fix

Use only fields declared on the struct. For type conflicts, rename the user type
or avoid defining a type with the same name as a required standard-library type.

## LSP Code Action

No automatic code action is currently provided.

# E0317: Duplicate Match Arm or Generic Struct Literal Needs Annotation

## Explanation

This diagnostic is used for two enum/struct lowering constraints:

- a `match` contains more than one arm for the same enum variant;
- a generic struct literal does not have enough type context to infer its type
  arguments.

## Error Example

```nomo
package app.main

enum Color {
    Red
}

fn label(color: Color) -> string {
    return match color {
        Color.Red => "red"
        Color.Red => "again"
    }
}
```

## Correct Code

```nomo
package app.main

enum Color {
    Red
}

fn label(color: Color) -> string {
    return match color {
        Color.Red => "red"
    }
}
```

For generic struct literals, add a type annotation:

```nomo
let box: Box<i64> = Box { value: 1 }
```

## Fix

Remove duplicate match arms. For generic struct literals, provide an expected
type through a variable annotation, return type, or call argument context.

## LSP Code Action

No automatic code action is currently provided.

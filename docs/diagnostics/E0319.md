# E0319: Match Initializer Has No Value-Producing Arm

## Explanation

When a `match` is used as an initializer, at least one arm must produce a value
so the initializer type can be determined. Arms that always diverge, such as
`panic(...)`, do not establish a value type.

## Error Example

```nomo
package app.main

enum Color {
    Red
}

fn main() -> void {
    let label: string = match Color.Red {
        Color.Red => panic("red")
    }
}
```

## Correct Code

```nomo
package app.main

enum Color {
    Red
}

fn main() -> void {
    let label: string = match Color.Red {
        Color.Red => "red"
    }
}
```

## Fix

Make at least one arm produce a value, or avoid using the match as an
initializer.

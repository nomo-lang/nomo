# E0324: Generic Enum Constructor Needs Annotation

## Explanation

A generic enum constructor was used without enough expected type context to
infer its type arguments. This commonly happens with unqualified prelude
variants such as `Some(...)` or `Ok(...)`.

## Error Example

```nomo
package app.main

fn main() -> void {
    let value = Some(1)
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    let value: Option<i32> = Some(1)
}
```

## Fix

Add a type annotation, return from a function with a concrete enum return type,
or pass the constructor to a call site with a known expected enum type.

## LSP Code Action

No automatic code action is currently provided.

# E0401: Invalid Main Signature

## Explanation

The project entry point must be `main` with no parameters. In v0.1 it may return
`void` or `Result<void, E>`, and it cannot be generic.

## Error Example

```nomo
package app.main

fn main() -> i32 {
    return 1
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
}
```

Or:

```nomo
package app.main

fn main() -> Result<void, string> {
    return Ok(void)
}
```

## Fix

Remove parameters and generic type parameters from `main`, and change its
return type to `void` or `Result<void, E>`.

## LSP Code Action

No automatic code action is currently provided.

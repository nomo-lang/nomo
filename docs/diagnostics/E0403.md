# E0403: Unsupported Type or Required Type Annotation

## Explanation

The type checker could not accept a type in a declaration or inference site.
This code is used for unsupported v0.1 type names, `void` in places that must
produce values, and `?` let bindings that omit the explicit unwrapped type.

For example, `int` is not a v0.1 builtin type. Use `i64` or an explicit-width
integer type instead.

## Error Example

```nomo
package app.main

fn main() -> void {
    let count: int = 1
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    let count: i64 = 1
}
```

For `?` bindings, keep the unwrapped type explicit:

```nomo
package app.main

fn parse() -> Result<i64, string> {
    return Ok(1)
}

fn compute() -> Result<i64, string> {
    let value: i64 = parse()?
    return Ok(value)
}
```

## Fix

Use a supported type name, import the standard-library type you reference, avoid
`void` in value-producing positions, or add an explicit non-void annotation to a
`let` binding that uses `?`.

## LSP Code Action

No automatic code action is currently provided.

# E0404: Type Mismatch

## Explanation

The type checker found a value whose actual type does not match the expected
type from an annotation, assignment target, return type, call argument, or
expression context.

## Error Example

```nomo
package app.main

fn main() -> void {
    let value: i32 = "bad"
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    let value: i32 = 1
}
```

## Fix

Use a value with the expected type, update the annotation, or add an explicit
cast when the conversion is supported.

## LSP Code Action

No automatic code action is currently provided.

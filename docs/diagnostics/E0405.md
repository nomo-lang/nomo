# E0405: Unsupported Value Expression

## Explanation

The parser accepted an expression shape, but the current v0.1 compiler reached a
value-lowering path that does not support that shape. This is mostly a compiler
guardrail: ordinary invalid source should usually receive a more specific
diagnostic such as an unknown name, type mismatch, unsupported `?` position, or
invalid `mut` argument.

## Error Example

There is no stable user-facing source pattern that should intentionally produce
this diagnostic. If normal source code reports E0405, please reduce the example
and file an issue because a more specific diagnostic is probably missing.

```nomo
package app.main

fn main() -> void {
    // A supported value expression should compile or report a more specific
    // diagnostic.
    let value: i64 = 1
}
```

## Correct Code

```nomo
package app.main

fn use_value(mut value: i64) -> void {
}

fn main() -> void {
    let value: i64 = 1
    use_value(value)
}
```

## Fix

Rewrite the expression into a supported value form such as a literal, variable,
call, struct literal, `match`, `if`, cast, or binary expression. If the source is
already a normal supported expression, treat E0405 as a missing compiler
diagnostic and report it.

## LSP Code Action

No automatic code action is currently provided.

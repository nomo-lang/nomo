# E0406: Missing Return Value

## Explanation

A function with a non-`void` return type must return a value on every path. The
compiler reports this when the lowered function body can fall through without
returning.

## Error Example

```nomo
package app.main

fn answer() -> i64 {
}

fn main() -> void {
}
```

## Correct Code

```nomo
package app.main

fn answer() -> i64 {
    return 42
}

fn main() -> void {
}
```

## Fix

Add a return statement or tail expression that produces the declared return
type on every control-flow path.

## LSP Code Action

No automatic code action is currently provided.

# E0420: Invalid Question Operator Carrier

## Explanation

The `?` operator can only unwrap Nomo's propagation carriers: `Result<T, E>`
and `Option<T>`. The type checker reports this error when the expression before
`?` has some other type, such as `i32`, `string`, a struct, or `void`.

## Error Example

```nomo
package app.main

fn parse() -> i32 {
    return 1
}

fn compute() -> Result<i32, string> {
    let value: i32 = parse()?
    return Ok(value)
}
```

## Correct Code

```nomo
package app.main

fn parse() -> Result<i32, string> {
    return Ok(1)
}

fn compute() -> Result<i32, string> {
    let value: i32 = parse()?
    return Ok(value)
}
```

## Fix

Use `?` only on calls or expressions that return `Result<T, E>` or `Option<T>`.
If the expression cannot fail or be absent, remove `?`. If it can fail, model
that in the function's return type.

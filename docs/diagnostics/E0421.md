# E0421: Incompatible Question Operator Propagation

## Explanation

The `?` operator returns early from the current function when it sees
`Result.Err(error)` or `Option.None`. The current function therefore must return
a compatible carrier:

- `Result<T, E>?` requires the current function to return `Result<U, E>`.
- `Option<T>?` requires the current function to return `Option<U>`.

This error is also used when the success wrapper around a `?` expression does
not match the carrier being propagated.

## Error Example

```nomo
package app.main

fn parse() -> Result<i32, string> {
    return Ok(1)
}

fn main() -> void {
    let value: i32 = parse()?
}
```

## Correct Code

```nomo
package app.main

fn parse() -> Result<i32, string> {
    return Ok(1)
}

fn compute() -> Result<i32, string> {
    let value: i32 = parse()?
    return Ok(value)
}

fn main() -> void {
}
```

## Fix

Change the current function to return a compatible `Result` or `Option`, handle
the carrier explicitly with `match`, or convert the error before `?` with an
explicit helper such as `std.result.map_err(...)`.

# E0422: Unsupported Question Operator Position

## Explanation

The current compiler lowers `?` by inserting an early-return check before the
expression that uses the unwrapped value. Most statement-level positions support
that rewrite, including `let`, assignment, `if`, `match`, `for`, `defer`, and
tail expressions.

This diagnostic is the remaining guard for a `?` expression that reaches value
lowering without one of those rewrites. For example, constants cannot perform
early returns, so `?` is not supported in a `const` initializer.

## Error Example

```nomo
package app.main

fn parse() -> Result<string, string> {
    return Ok("value")
}

const VALUE: string = parse()?
```

## Correct Code

```nomo
package app.main

fn parse() -> Result<string, string> {
    return Ok("value")
}

fn compute() -> Result<string, string> {
    let value: string = parse()?
    return Ok(value)
}
```

## Fix

Move the `?` expression into a statement-level function body where early return
is meaningful. For constants, store a non-fallible value or compute the fallible
value inside a function that returns `Result` or `Option`.

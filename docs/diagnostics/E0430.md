# E0430: Non-Constant Const Initializer

## Explanation

Top-level `const` initializers must be constant expressions in the current v0.1
implementation. Today that means a literal value.

## Error Example

```nomo
package app.main

fn one() -> i32 {
    return 1
}

const X: i32 = one()
```

## Correct Code

```nomo
package app.main

const X: i32 = 1
```

## Fix

Use a literal initializer for `const`, or compute the value inside a function
with `let`.

# E0501: Immutable Value Mutation

## Explanation

Nomo bindings are immutable by default. This error is reported when code tries
to assign through, update, or mutably borrow a binding or field that was not
declared mutable.

## Error Example

```nomo
package app.main

fn main() -> void {
    let value: i64 = 1
    value = 2
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    let mut value: i64 = 1
    value = 2
}
```

## Fix

Declare the binding as `mut` only when mutation is intentional. Otherwise,
introduce a new immutable binding with the desired value.

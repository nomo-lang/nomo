# E0502: Conflicting Mutable Borrows

## Explanation

A single call tried to create overlapping mutable borrows. Nomo permits
non-overlapping mutable field borrows, but it rejects borrowing the same value
twice, or borrowing both a value and one of its fields, in the same call.

## Error Example

```nomo
package app.main

fn combine(mut left: i64, mut right: i64) -> i64 {
    return left + right
}

fn main() -> void {
    let mut count: i64 = 41
    let answer: i64 = combine(mut count, mut count)
}
```

## Correct Code

```nomo
package app.main

fn combine(mut left: i64, mut right: i64) -> i64 {
    return left + right
}

fn main() -> void {
    let mut left: i64 = 41
    let mut right: i64 = 1
    let answer: i64 = combine(mut left, mut right)
}
```

## Fix

Avoid overlapping mutable borrows in one call. Split the operation into
multiple statements, use distinct mutable bindings, or pass non-overlapping
fields when the callee accepts those field types.

No automatic code action is currently provided.

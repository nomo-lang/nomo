# E0521: Non-Diverging Let-Else Body

## Explanation

The `else` body in a `let ... else` statement must diverge. In v0.1, that means
it must leave the current path with `panic`, `return`, `break`, or `continue`.
If the `else` body can fall through, the binding after `else` would not be
initialized.

## Error Example

```nomo
package app.main

fn main() -> void {
    let value: Option<i32> = None
    let Some(number) = value else {
        let fallback: i32 = 0
    }
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    let value: Option<i32> = None
    let Some(number) = value else {
        panic("missing number")
    }
}
```

## Fix

Make the `else` block diverge with `panic`, `return`, `break`, or `continue`, or
rewrite the logic as `if let`/`match` when both paths should continue.

No automatic code action is currently provided.

# E0800: Invalid Standard-Library Intrinsic Manifest

## Explanation

The toolchain-owned `std/intrinsics.toml` does not describe the canonical
standard-library identities expected by the compiler. The manifest is a
toolchain file; user packages cannot replace or override it.

## Error Example

```toml
schema = 1
package = "custom/std"
```

## Correct Code

```toml
schema = 1
package = "nomo-lang/std"
```

Keep required `Option`, `Result`, and postfix `?` bindings unique, point each
binding at its matching `std/src/*.nomo` module, and use a supported binding
kind.

## Fix

Restore the canonical manifest shipped with the Nomo toolchain. Do not add a
user dependency or a `#[lang]` declaration to work around this diagnostic.

## LSP Code Action

No automatic code action is currently provided.

# E0901: Manifest or Project Configuration Error

## Explanation

Project-level tooling could not parse or validate the nearest `nomo.toml`, or
could not build a module context from the project configuration.

## Error Example

```toml
[dependencies]
std = { package = "custom/std", path = "../std" }
```

## Correct Code

```toml
[package]
namespace = "local"
name = "hello"
version = "0.1.0"
edition = "2026"
```

## Fix

Use standard TOML syntax, keep package metadata valid, and do not override the
reserved `std` import root with a dependency alias.

## LSP Code Action

No automatic code action is currently provided.

# E0902: Project Source Processing Error

## Explanation

Project tooling could not process a `.nomo` source file. This can happen while
formatting a project, collecting project symbols, loading an imported module
from disk, or rejecting a construct the formatter cannot safely rewrite.

## Error Example

```nomo
package app.main

import app.generated

fn main() -> void {
}
```

If `src/generated.nomo` exists in the module graph but cannot be read by the
tooling process, the project command reports `E0902`.

## Correct Code

```nomo
package app.main

import app.generated

fn main() -> void {
}
```

with a readable module file:

```text
src/generated.nomo
```

## Fix

Check that the file or directory exists, is readable, and is not blocked by
filesystem permissions. For formatter-specific `defer` errors, rewrite the
`defer` body as an expression statement before running `nomo fmt`.

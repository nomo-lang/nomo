# E0904: Module Package Mismatch

## Explanation

An imported module file was found, but its `package` declaration does not match
the import path used to load it. The package path inside the file must describe
the module path.

For example, `import app.math` requires the loaded file to declare
`package app.math`.

## Error Example

```nomo
package app.other

pub fn add(a: i64, b: i64) -> i64 {
    return a + b
}
```

when the file is loaded through:

```nomo
import app.math
```

## Correct Code

```nomo
package app.math

pub fn add(a: i64, b: i64) -> i64 {
    return a + b
}
```

## Fix

Update the module file's `package` declaration to match the import path, or
rename and move the file so the package path and file mapping agree.

## LSP Code Action

Yes. When `nomo-lsp` can infer the expected module path from the current file
location, it exposes a quick fix that updates the `package` declaration.

# E1519: Invalid FFI or Unsafe Boundary

## Explanation

The source uses an FFI or unsafe boundary in a shape the current compiler does
not support. This includes unsafe blocks that contain more than one statement,
plain `string` extern parameters, `CString` extern return values, and extern
calls with invalid arguments.

## Error Example

```nomo
package app.main

import std.ffi

extern "C" {
    fn puts(message: CString) -> i32
}

fn main() -> void {
    let first: CString = CString.from_string("one")
    let second: CString = CString.from_string("two")
    unsafe {
        puts(first)
        puts(second)
    }
}
```

## Correct Code

```nomo
package app.main

import std.ffi

extern "C" {
    fn puts(message: CString) -> i32
}

fn main() -> void {
    let first: CString = CString.from_string("one")
    let second: CString = CString.from_string("two")
    unsafe {
        puts(first)
    }
    unsafe {
        puts(second)
    }
}
```

## Fix

Keep each unsafe block to one supported statement. Import `std.ffi`, convert
Nomo strings with `CString.from_string`, use `Opaque` only as an uninterpreted
handle, and make sure extern calls use supported argument and return types.

## LSP Code Action

No automatic code action is currently provided.

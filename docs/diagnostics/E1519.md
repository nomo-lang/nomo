# E1519: Invalid FFI or Unsafe Boundary

## Explanation

The source uses a v0.1 FFI or unsafe boundary in a shape the current compiler
does not support. Today this covers unsafe blocks that contain more than one
statement and supported extern calls with invalid arguments.

## Error Example

```nomo
package app.main

fn main() -> void {
    unsafe {
        puts("one")
        puts("two")
    }
}
```

## Correct Code

```nomo
package app.main

fn main() -> void {
    unsafe {
        puts("one")
    }
    unsafe {
        puts("two")
    }
}
```

## Fix

Keep each v0.1 unsafe block to one supported statement, and make sure extern
calls use the supported argument count and types.

## LSP Code Action

No automatic code action is currently provided.

# E1521: Conflicting Opaque Handle Type

## Explanation

An opaque handle name conflicts with a built-in, concrete, or previously
declared type.

## Error Example

```nomo
extern opaque type Window
extern opaque type Window
```

## Correct Code

```nomo
extern opaque type Window
extern opaque type Surface
```

## Fix

Give every opaque handle a unique name and do not reuse names such as `CString`,
`Opaque`, or a struct/enum already declared in the package.

## LSP Code Action

No automatic code action is currently provided.

# E1525: Callback Is Not ABI-Safe

## Explanation

An extern C callback uses a parameter or return type outside the supported C
ABI surface, or is passed somewhere other than an extern function parameter.

## Error Example

```nomo
extern "C" {
    fn register(callback: extern "C" fn(string) -> void) -> void
}
```

## Correct Code

```nomo
extern "C" {
    fn register(callback: extern "C" fn(i32) -> void) -> void
}
```

## Fix

Use fixed-width scalars, handles, nullable handles, supported C records, or
`void` returns. Keep callback values as arguments to extern C functions.

## LSP Code Action

No automatic code action is currently provided.
